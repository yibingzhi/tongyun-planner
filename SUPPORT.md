## 报告问题

### Issue 模板
我们提供了以下 Issue 模板：
- **Bug 报告** - 报告软件缺陷
- **功能请求** - 提出新功能建议
- **文档问题** - 报告文档错误或改进建议
- **性能问题** - 报告性能相关问题
- **安全问题** - 报告安全漏洞（请通过邮件私下报告）

### 报告指南
为了更好地帮助您，请在报告问题时提供：
1. 清晰的问题描述
2. 详细的重现步骤
3. 期望的行为
4. 实际的行为
5. 环境信息（操作系统、软件版本等）
6. 截图或错误日志（如果适用）

## 功能请求

### 请求流程
1. 在讨论区搜索是否已有类似请求
2. 如果没有，请创建新的功能请求 Issue
3. 详细描述功能需求和使用场景
4. 等待社区反馈和维护者评估
5. 如果功能被接受，欢迎贡献代码

### 优先级评估
我们会根据以下因素评估功能优先级：
- 用户需求程度
- 实现复杂度
- 与项目愿景的一致性
- 社区投票结果

## 贡献代码

### 贡献流程
1. 阅读 [CONTRIBUTING.md](CONTRIBUTING.md) 文件
2. Fork 项目仓库
3. 创建功能分支
4. 实现功能并测试
5. 提交 Pull Request
6. 等待代码审查

### 代码审查
- 所有 Pull Request 都需要经过代码审查
- 审查内容包括代码质量、功能正确性、测试覆盖等
- 审查通过后会合并到主分支
