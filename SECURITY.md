## 报告漏洞

如果您发现安全漏洞，请按照以下步骤报告：

### 1. 不要公开披露
请不要在公开的 Issue 中报告安全漏洞。

### 2. 发送邮件
请发送邮件到 [security@qiyunlist.com]（请替换为实际邮箱），包含：
- 漏洞描述
- 重现步骤
- 影响范围
- 建议的修复方案

### 3. 响应时间
- 我们会在 48 小时内确认收到报告
- 我们会在 7 天内提供初步评估
- 我们会在 30 天内提供修复方案

### 4. 保密性
我们会与您合作，在修复发布前保持漏洞的保密性。

## 安全更新

### 更新频率
- 严重漏洞：立即修复
- 高危漏洞：7 天内修复
- 中危漏洞：30 天内修复
- 低危漏洞：下一版本修复

### 更新方式
- 安全更新会通过常规版本发布
- 严重漏洞会发布紧急更新
- 更新日志会包含安全修复说明

## 安全最佳实践

### 用户数据保护
- 所有用户数据存储在本地
- 不收集用户个人信息
- 不跟踪用户行为
- 不上传用户数据

### API 密钥安全
- API 密钥存储在本地
- 不上传到服务器
- 支持加密存储（计划中）
- 定期更换密钥建议

### 网络通信
- 使用 HTTPS 协议
- 支持 WebDAV 加密连接
- 不收集使用数据
- 不跟踪用户行为

### 应用安全
